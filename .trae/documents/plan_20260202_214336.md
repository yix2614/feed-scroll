## 问题定位
用户把 `blur` 从 3 改成 20（或任意值）后，页面上玻璃模糊**完全没有变化**，疑似样式未更新或浏览器未重新计算。

## 检查项
1. HMR 是否成功：确认 vite 终端无报错，且页面 console 无热更新失败日志。
2. 元素样式：在 DevTools → Elements 选中下载卡片，确认实时 `backdrop-filter: blur(...px)` 是否已变为新值。
3. 浏览器缓存：强制刷新（Cmd+Shift+R）或无痕窗口再试，排除缓存。
4. 硬件加速：检查是否因 transform/opacity 优化导致模糊被跳过。

## 修复方案
- 若样式值已变但肉眼无差异 → 把最小不透明度再提高到 0.25 并加 `will-change: backdrop-filter` 强制重绘。
- 若样式值没变 → 检查组件是否未重新渲染，或样式对象被缓存；改用 key 强制重挂载。

## 验证
改完再次调整 `blur`，保存后应立即看到模糊强度变化。
